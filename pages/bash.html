Bash Topics

Because I'm always forgetting certain commands in bash, I made myself a list of the commands I keep having to look up.

Quick commands to get around the line you're typing in bash.

  • ctrl-A Goes to start of line
  • ctrl-E Goes to end of line
  • alt (or option) -F Goes forward 1 word
  • alt (or option ) -B Goes back 1 word
  • ctrl-U Deletes from cursor to start of line
  • ctrl-K Deletes from cursor to end of line
  • alt-D Deletes from cursor to end of word
  • ctrl-XX Toggle between current cursor position and start of line

Access your bash history

  • history will give you a line numbered list of all the commands you have entered for some period of time, dependent on your system's settings. If you are working in multiple terminal windows, I don't know how to keep track of what's what.
  • !132 To run the same command as line 132 (or whatever line) again
  • history | grep "phrase to search" to find lines in your history with that phrase
  • history 5 Display just the last 5 (or however many) lines in your history

From the bash shell, you can find files or directories by name. You can also find files by text contents.

Finding files by name

  • find ~/path/to/files/ -name "thesis*.*" -type f will find all files with names that start with thesis in the specified directory. Use type f for files and -type d for directories.

    • Other flags include:

  • -mtime -1 Modified in the last 1 day
  • -mmin -30 Modfied in the last 30 minutes
  • -size +1024M Files over 1 gigabyte
  • ! -name "*.pdf" Exclude files with pdf extension
  • -delete Delete the results Do not do this; this is irreversible!

Finding files by text content

  • grep -r "phrase to search" . will find files that contain that phrase. The -r flag says to search recursively (files inside folders) and the . says to look in the current directory.

    • Other flags include:

  • -l Display only the filename and path in the results. Otherwise it displays the filename and path along with the target text bounded by a few * characters.
  • -w Set word boundaries, so it will find only the word surrounded by whitespace. So grep -r java . will find javascript while grep -rw java . will not.
  • -n Displays line numbers of where the search term appears in the document in the results.
  • -i Makes the search case insensitive.
  • -o Display only the target text in the results.
  • -E Use regular expressions in the search term.
  • --exclude-dir=dirA Exclude dirA in the search results.
  • --exclude-dir={dirA,dirB} Exclude dirA and dirB in the search results. This only works for two or more directory nanes. No spaces anywhere in that line.
  • -E Use regular expressions in the search term..

Examples

grep -rnoE '.{0,3}target.{0,10}' . This will find the word 'target' and display the 3 characters before it and the 10 characters after it.

grep -rnoE '.{0,3}\bb[a-z]+r\b.{0,10}' . This will find any lowercase word that starts with b and ends with r.
